Mes commandes utiles

Une fiche simple pour réviser Git, Linux, CMake, GCC, tests, SDL2, HTML et LaTeX. Tu peux chercher un mot comme coverage, switch, cp -r, add_test ou includegraphics.

Git

git clone <url>
Cloner un dépôt Git distant sur ta machine.
git status
Voir l’état actuel du dépôt.
git add <fichier>
Ajouter un fichier à l’index.
git add .
Ajouter toutes les modifications du dossier courant.
git commit -m "message explicite"
Créer un commit avec un message clair.
git log
Afficher l’historique des commits.
git diff
Voir les différences non commit.
git pull
Récupérer les modifications distantes.
git push
Envoyer les commits vers le dépôt distant.
git branch
Afficher les branches locales.
git branch -a
Afficher les branches locales et distantes.
git switch <branche>
Changer de branche.
Très important
git switch -c <nouvelle_branche>
Créer puis ouvrir une nouvelle branche.
git checkout <branche>
Ancienne commande pour changer de branche.
git merge <branche>
Fusionner une branche dans la branche courante.
git reset --hard HEAD~1
Supprimer le dernier commit et revenir à l’état précédent.
Question fréquente
git fetch
Récupérer les mises à jour distantes sans fusion immédiate.
git remote -v
Voir les URL des dépôts distants.

Configuration Git

git config --global user.name "Prénom Nom"
Définir le nom d’auteur global.
git config --global user.email "prenom.nom@etu.u-bordeaux.fr"
Définir l’email global.
git config --global core.editor "nano"
Définir l’éditeur par défaut.
git config --global pull.rebase false
Configurer le comportement de git pull.
git config --global -l
Afficher toute la configuration Git globale.

SSH / GitLab

ssh-keygen
Créer une clé SSH.
cat ~/.ssh/id_rsa.pub
Afficher la clé publique à copier dans GitLab.
ssh -T git@gitlab.emi.u-bordeaux.fr
Tester la connexion SSH à GitLab.

Linux / terminal

pwd
Afficher le dossier courant.
ls
Lister les fichiers.
ls -l
Lister avec détails.
ls -la
Lister tout, y compris les fichiers cachés.
cd <dossier>
Entrer dans un dossier.
mkdir <dossier>
Créer un dossier.
mkdir -p <dossier>
Créer un dossier et ses parents si besoin.
cp <fichier> <destination>
Copier un fichier.
cp -r <dossier> <destination>
Copier un répertoire.
Question fréquente
mv <source> <destination>
Déplacer ou renommer.
rm <fichier>
Supprimer un fichier.
rm -r <dossier>
Supprimer un dossier récursivement.
rm -rf build
Supprimer complètement le dossier build.
find . -name "*.c"
Chercher les fichiers .c.
grep -R "mot" .
Chercher un mot dans les fichiers.
cat <fichier>
Afficher un fichier dans le terminal.
echo $?
Voir le code de retour de la dernière commande.

CMake / build

mkdir build
Créer le dossier de compilation.
cd build
Entrer dans le dossier build.
cmake ..
Générer les fichiers de build out-of-source.
cmake .
Générer les fichiers de build in-source.
make
Compiler le projet.
make VERBOSE=ON
Voir les vraies commandes de compilation.
make test
Lancer les tests.
ctest
Lancer tous les tests avec CTest.
ctest -R dummy
Lancer seulement les tests qui correspondent au motif.
make ExperimentalCoverage
Calculer la couverture du code.
make ExperimentalMemCheck
Chercher les fuites mémoire avec Valgrind via CMake.

Fonctions CMake à connaître

cmake_minimum_required(VERSION 3.5)
Version minimale de CMake.
project(monprojet C)
Définir le projet.
set(CMAKE_C_FLAGS "-std=c99 -g -Wall")
Définir les options de compilation C.
add_library(game game.c game_aux.c)
Créer une bibliothèque.
add_executable(game_text game_text.c)
Créer un exécutable.
target_link_libraries(game_text game m)
Lier une bibliothèque à un exécutable.
Question fréquente
enable_testing()
Activer les tests.
include(CTest)
Ajouter les cibles liées à CTest et MemCheck.
add_test(test_dummy ./game_test_login dummy)
Ajouter un test CTest.
Question fréquente

GCC

-g
Ajouter les infos de debug.
-Wall
Afficher les warnings principaux.
-Wextra
Afficher encore plus de warnings.
-std=c99
Compiler en norme C99.
-O0
Pas d’optimisation.
-O1
Optimisation légère.
-O2
Optimisation avancée.
-O3
Niveau maximal d’optimisation.
Question fréquente
--coverage
Activer la couverture de code.
Question fréquente
gcc -std=c99 -g -Wall main.c -o prog
Compilation simple.
gcc -std=c99 -g -Wall main.c lib.c -lm -o prog
Compilation avec la bibliothèque mathématique.

Tests / debug / mémoire

./test_queue length
Lancer un test précis.
echo $?

valgrind ./prog
Exécuter un programme sous Valgrind.
valgrind --leak-check=full ./prog
Analyse détaillée des fuites mémoire.
